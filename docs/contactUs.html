Reporting tools

Bugs should be reported by using the Apache Bug Database.
Before reporting a problem, please check the database and the project history file to see if the problem is already known.

Support requests should be sent to the log4j-user list (see below).


Mailing lists

New releases and important bug-fixes are announced at the log4j-announce list. You can subscribe to the log4j-announce list, or change your existing subscription online. The traffic on this list is extremely low and you are encouraged to subscribe to this list if you are using log4j.

Discussion on log4j and logging in general are held on the log4j-user mailing list. This list is archived. To post a message to all its members, send e-mail to log4j-user@jakarta.apache.org. To subscribe to the log4j-users list send an e-mail to log4j-user-subscribe@jakarta.apache.org.

Development related discussions are held on the log4j-dev mailing list. This list is archived. To post a message to all its members, send e-mail to log4j-dev@jakarta.apache.org. To subscribe to the log4j-dev list send an e-mail to log4j-dev-subscribe@jakarta.apache.org.

All developers are encouraged to subscribe to the log4j-cvs list which will notify you of CVS commits to the repository. This list is archived. To post a message to all its members, send e-mail to log4j-cvs@jakarta.apache.org. To subscribe to the log4j-cvs list send an e-mail to log4j-cvs-subscribe@jakarta.apache.org.
